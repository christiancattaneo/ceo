Memory Systems for Long-Running Claude Agents

The four memory types agents need — working, episodic, semantic, and procedural — with Python implementations for each, memory compression to stay within context limits, and Redis-backed cross-session persistence.

Apr 14, 2026  ·  12 min read  ·  CEO Editorial

A single Claude conversation is stateless between API calls. Everything the model "knows" must fit in the context window on each request. For short interactions this is fine; for agents that operate over hours, days, or many sessions, naive approaches — dumping the full conversation history into every call — hit context limits fast and become expensive.

The solution is a layered memory architecture that mirrors how humans store information: immediate working memory for the current task, episodic memory for past events, semantic memory for knowledge and facts, and procedural memory for skills and tools.

1. The Four Memory Types

Type What it stores Retrieval Backing store
Working memory Current conversation — the messages list for the active call Always in context In-process list
Episodic memory Past events: actions taken, results observed, decisions made Recent N entries or semantic search Redis / SQLite / Postgres
Semantic memory Facts, knowledge, user preferences, world state Vector similarity search pgvector / Pinecone / Chroma
Procedural memory Available tools, their signatures, and when to use them Static injection or dynamic lookup by task type In-process registry

2. Working Memory — Conversation History

Working memory is the messages list. The challenge is keeping it bounded. Use a sliding window with compression: when the history grows past a token threshold, summarize the oldest turns and replace them with the summary.

from dataclasses import dataclass, field
import anthropic

client = anthropic.Anthropic()

@dataclass
class WorkingMemory:
    messages: list[dict] = field(default_factory=list)
    system: str = ""
    max_tokens: int = 16_000  # compress when history exceeds this
    model: str = "claude-sonnet-4-6"

    def add(self, role: str, content: str) -> None:
        self.messages.append({"role": role, "content": content})
        self._maybe_compress()

    def _token_count(self) -> int:
        result = client.messages.count_tokens(
            model=self.model,
            system=self.system,
            messages=self.messages,
        )
        return result.input_tokens

    def _maybe_compress(self) -> None:
        if self._token_count() < self.max_tokens:
            return

        # Summarize the oldest half of the conversation
        split = len(self.messages) // 2
        old_turns = self.messages[:split]
        self.messages = self.messages[split:]

        old_text = "\n".join(
            f"{m['role']}: {m['content']}" for m in old_turns
        )
        summary_response = client.messages.create(
            model="claude-haiku-4-5-20251001",
            max_tokens=512,
            system="Summarize this conversation excerpt concisely, preserving key facts, decisions, and context needed to continue the conversation.",
            messages=[{"role": "user", "content": old_text}]
        )
        summary = summary_response.content[0].text

        # Prepend summary as a system injection
        self.messages.insert(0, {
            "role": "user",
            "content": f"[Prior context summary]\n{summary}"
        })
        self.messages.insert(1, {
            "role": "assistant",
            "content": "Understood, I have the prior context."
        })

3. Episodic Memory — Event Store

Episodic memory records what the agent did and observed — a durable log of past sessions. On each new session, the agent can load recent episodes to understand what it was doing and what it learned.

import json, time, sqlite3
from dataclasses import dataclass, asdict

@dataclass
class Episode:
    session_id: str
    timestamp: float
    action: str           # what the agent did
    observation: str      # what it observed / result
    importance: float     # 0.0–1.0, set by agent
    tags: list[str]

class EpisodicMemory:
    def __init__(self, db_path: str = "agent_memory.db"):
        self.conn = sqlite3.connect(db_path)
        self._init_schema()

    def _init_schema(self) -> None:
        self.conn.execute("""
            CREATE TABLE IF NOT EXISTS episodes (
                id INTEGER PRIMARY KEY AUTOINCREMENT,
                session_id TEXT,
                timestamp REAL,
                action TEXT,
                observation TEXT,
                importance REAL,
                tags TEXT
            )
        """)
        self.conn.commit()

    def store(self, episode: Episode) -> None:
        self.conn.execute(
            "INSERT INTO episodes (session_id, timestamp, action, observation, importance, tags) VALUES (?,?,?,?,?,?)",
            (episode.session_id, episode.timestamp, episode.action,
             episode.observation, episode.importance, json.dumps(episode.tags))
        )
        self.conn.commit()

    def recall_recent(self, n: int = 10, min_importance: float = 0.5) -> list[Episode]:
        rows = self.conn.execute(
            "SELECT session_id, timestamp, action, observation, importance, tags FROM episodes WHERE importance >= ? ORDER BY timestamp DESC LIMIT ?",
            (min_importance, n)
        ).fetchall()
        return [
            Episode(r[0], r[1], r[2], r[3], r[4], json.loads(r[5]))
            for r in rows
        ]

    def format_for_context(self, episodes: list[Episode]) -> str:
        if not episodes:
            return "No prior episodes."
        lines = ["Recent episodes (newest first):"]
        for ep in episodes:
            ts = time.strftime("%Y-%m-%d %H:%M", time.localtime(ep.timestamp))
            lines.append(f"[{ts}] {ep.action} → {ep.observation}")
        return "\n".join(lines)
Importance scoring Have the agent score each episode's importance (0–1) before storing it. A tool call that returned an error is high importance (0.9); a routine status check is low (0.2). Filtering by importance at recall time keeps the context injection concise. You can also let Claude score importance: one Haiku call at end-of-episode costs fractions of a cent.

4. Semantic Memory — Fact Store

Semantic memory stores facts the agent has learned — user preferences, domain knowledge, world state that shouldn't be re-derived each session. It's a small vector store the agent can query by semantic similarity.

import voyageai
import numpy as np
from dataclasses import dataclass

voyage = voyageai.Client()

@dataclass
class Fact:
    content: str
    source: str          # where this fact came from
    confidence: float    # 0–1
    embedding: list[float] = None

class SemanticMemory:
    def __init__(self):
        self.facts: list[Fact] = []

    def store_fact(self, content: str, source: str, confidence: float = 1.0) -> None:
        embedding = voyage.embed([content], model="voyage-3", input_type="document").embeddings[0]
        self.facts.append(Fact(content=content, source=source, confidence=confidence, embedding=embedding))

    def recall(self, query: str, k: int = 5, min_confidence: float = 0.6) -> list[Fact]:
        if not self.facts:
            return []

        q_emb = np.array(voyage.embed([query], model="voyage-3", input_type="query").embeddings[0])
        eligible = [f for f in self.facts if f.confidence >= min_confidence]

        scored = []
        for fact in eligible:
            f_emb = np.array(fact.embedding)
            sim = float(np.dot(q_emb, f_emb) / (np.linalg.norm(q_emb) * np.linalg.norm(f_emb)))
            scored.append((sim, fact))

        scored.sort(reverse=True, key=lambda x: x[0])
        return [f for _, f in scored[:k]]

    def update_fact(self, old_content: str, new_content: str, source: str) -> None:
        """Replace an existing fact (agents can correct their beliefs)."""
        self.facts = [f for f in self.facts if f.content != old_content]
        self.store_fact(new_content, source)

5. Procedural Memory — Tool Registry

Procedural memory is the agent's knowledge of what it can do. A static tool list works for simple agents; for complex ones, a registry that selects tools by task type avoids flooding the context with irrelevant tool definitions.

from dataclasses import dataclass
from typing import Callable

@dataclass
class Tool:
    name: str
    description: str
    input_schema: dict
    handler: Callable
    tags: list[str]  # e.g. ["search", "web", "research"]

class ProceduralMemory:
    def __init__(self):
        self._tools: dict[str, Tool] = {}

    def register(self, tool: Tool) -> None:
        self._tools[tool.name] = tool

    def get_for_task(self, task_tags: list[str]) -> list[dict]:
        """Return API-formatted tool definitions relevant to the task."""
        relevant = [
            t for t in self._tools.values()
            if any(tag in t.tags for tag in task_tags)
        ]
        return [
            {"name": t.name, "description": t.description, "input_schema": t.input_schema}
            for t in relevant
        ]

    def execute(self, name: str, inputs: dict) -> str:
        tool = self._tools.get(name)
        if not tool:
            raise ValueError(f"Unknown tool: {name}")
        return tool.handler(**inputs)

6. Assembling the Full Memory Context

At the start of each agent call, assemble relevant memories into the system prompt. Order matters: procedural context (what tools exist) first, then semantic facts (what the agent knows), then episodic summary (what it did recently).

def build_agent_context(
    base_system: str,
    query: str,
    episodic: EpisodicMemory,
    semantic: SemanticMemory,
    procedural: ProceduralMemory,
    task_tags: list[str],
) -> tuple[str, list[dict]]:
    """Returns (system_prompt, tools) with memory injected."""

    # Episodic: last 5 important episodes
    recent_eps = episodic.recall_recent(n=5, min_importance=0.6)
    episodic_block = episodic.format_for_context(recent_eps)

    # Semantic: facts relevant to this query
    relevant_facts = semantic.recall(query, k=4)
    semantic_block = "\n".join(
        f"- {f.content} (confidence: {f.confidence:.1f})"
        for f in relevant_facts
    ) or "None relevant."

    system = f"""{base_system}

## What you know
{semantic_block}

## Recent actions
{episodic_block}"""

    # Procedural: tools relevant to this task
    tools = procedural.get_for_task(task_tags)

    return system, tools
Memory injection budget Memory context competes with user input and response tokens for the context window. Budget explicitly: episodic block ≤ 500 tokens, semantic block ≤ 300 tokens, tool definitions ≤ 1,000 tokens. Use count_tokens to verify before sending. If a block would exceed budget, trim to the highest-importance items.

Production Checklist

  • Working memory compresses when token count exceeds threshold — never truncates hard
  • Episodic events scored for importance before storage; recall filters by importance threshold
  • Semantic facts include source and confidence — agent can update or retract beliefs
  • Procedural tool registry selects tools by task type — full tool list never injected blindly
  • Memory injection is token-budgeted — each block has an explicit max
  • Episodic and semantic stores backed by durable storage (SQLite / Postgres) for cross-session persistence
  • Compression uses Haiku (cheap) not Sonnet — meta-tasks don't need frontier reasoning
  • Memory context assembled once per agent turn, not rebuilt for each sub-call in a multi-agent pipeline
  • Tests verify agent behavior across session boundaries — memory retrieval is tested, not just storage
  • Stale semantic facts have a TTL or explicit invalidation path — agents don't act on outdated beliefs indefinitely
