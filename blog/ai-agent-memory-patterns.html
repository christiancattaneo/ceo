AI Agent Memory Patterns: Episodic, Semantic, and Procedural Memory for LLMs

An LLM with no memory is stateless by design — every call starts fresh. That's fine for a one-shot summarizer, but it's wrong for any agent that needs to know what happened in a prior session, what the user told it last week, or how to handle a class of problem it's seen before. Implementing memory for agents is a distinct engineering problem, and getting it wrong produces agents that are expensive, inconsistent, and frustrating to use.

The Memory Taxonomy

Cognitive science distinguishes three types of long-term memory that map cleanly onto the problems production AI agents face:

Memory Type What It Stores Agent Analogue Primary Storage
Episodic Specific past events — "what happened when" Conversation history, past interactions, user actions Vector DB (by embedding) or key-value store
Semantic General facts about the world and the user User preferences, domain knowledge, entity attributes Structured DB + vector search
Procedural How to do things — learned skills and workflows Few-shot examples, tool usage patterns, learned heuristics Prompt store, fine-tuned model, example DB

Most production agents need all three. An agent that can recall past conversations (episodic) but doesn't remember that the user prefers concise answers (semantic) will consistently frustrate. An agent with rich episodic and semantic memory but no procedural memory will make the same mistakes repeatedly.

In-Context vs. External Memory

The simplest memory implementation is the conversation history in the context window. Append every message, pass the full history with each call, done. This works until it doesn't: at scale, it's expensive, slow, and eventually impossible as the window fills.

from dataclasses import dataclass, field
from typing import Literal
import time


MessageRole = Literal["user", "assistant", "system"]


@dataclass
class Message:
    role: MessageRole
    content: str
    timestamp: float = field(default_factory=time.time)
    tokens: int = 0


class InContextMemory:
    """Naive: keep everything in the context window."""

    def __init__(self, max_tokens: int = 16_000):
        self.messages: list[Message] = []
        self.max_tokens = max_tokens

    def add(self, role: MessageRole, content: str, tokens: int = 0) -> None:
        self.messages.append(Message(role=role, content=content, tokens=tokens))
        self._evict_if_needed()

    def _evict_if_needed(self) -> None:
        total = sum(m.tokens for m in self.messages)
        while total > self.max_tokens and len(self.messages) > 1:
            # Drop oldest non-system message
            for i, m in enumerate(self.messages):
                if m.role != "system":
                    removed = self.messages.pop(i)
                    total -= removed.tokens
                    break

    def as_api_messages(self) -> list[dict]:
        return [{"role": m.role, "content": m.content} for m in self.messages]

In-context memory has a hard ceiling. The real solution is external memory with selective retrieval — pull only what's relevant to the current turn, not everything that ever happened.

Episodic Memory: Storing and Retrieving Past Events

Episodic memory stores specific interactions as retrievable records. Each record is embedded and stored in a vector database. At query time, you compute the embedding of the current context and retrieve the K most similar past episodes to inject into the context window.

import anthropic
import numpy as np
from dataclasses import dataclass, field
from typing import Any
import json
import time
import hashlib


@dataclass
class Episode:
    id: str
    session_id: str
    user_id: str
    content: str                    # what was said/done
    summary: str                    # compressed version for retrieval
    embedding: list[float]
    metadata: dict[str, Any]
    created_at: float = field(default_factory=time.time)
    importance: float = 1.0         # 0-1, used for weighted retrieval


def episode_id(user_id: str, content: str) -> str:
    return hashlib.sha256(f"{user_id}:{content}:{time.time()}".encode()).hexdigest()[:16]


class EpisodicMemoryStore:
    def __init__(self, client: anthropic.Anthropic, embedding_model: str = "voyage-3"):
        self.client = client
        self.embedding_model = embedding_model
        self.episodes: list[Episode] = []   # replace with vector DB in production

    def embed(self, text: str) -> list[float]:
        # Pseudocode — substitute your embedding provider (Voyage, OpenAI, Cohere, etc.)
        response = self.client.beta.messages  # placeholder
        # return voyage_client.embed([text]).embeddings[0]
        return [0.0] * 1024   # stub

    def store(
        self,
        user_id: str,
        session_id: str,
        content: str,
        metadata: dict | None = None,
        importance: float = 1.0,
    ) -> Episode:
        summary = self._summarize(content)
        embedding = self.embed(summary)
        ep = Episode(
            id=episode_id(user_id, content),
            session_id=session_id,
            user_id=user_id,
            content=content,
            summary=summary,
            embedding=embedding,
            metadata=metadata or {},
            importance=importance,
        )
        self.episodes.append(ep)
        return ep

    def _summarize(self, content: str) -> str:
        if len(content) < 200:
            return content
        result = self.client.messages.create(
            model="claude-haiku-4-5-20251001",
            max_tokens=100,
            messages=[{
                "role": "user",
                "content": f"Summarize in 1-2 sentences for memory retrieval:\n\n{content}"
            }],
        )
        return result.content[0].text

    def retrieve(
        self,
        query: str,
        user_id: str,
        k: int = 5,
        min_importance: float = 0.0,
    ) -> list[Episode]:
        query_emb = np.array(self.embed(query))
        scored = []
        for ep in self.episodes:
            if ep.user_id != user_id:
                continue
            if ep.importance < min_importance:
                continue
            ep_emb = np.array(ep.embedding)
            # Cosine similarity weighted by importance
            sim = float(np.dot(query_emb, ep_emb) /
                        (np.linalg.norm(query_emb) * np.linalg.norm(ep_emb) + 1e-8))
            scored.append((sim * ep.importance, ep))
        scored.sort(key=lambda x: x[0], reverse=True)
        return [ep for _, ep in scored[:k]]

    def format_for_context(self, episodes: list[Episode]) -> str:
        if not episodes:
            return ""
        lines = ["Relevant past context:"]
        for ep in episodes:
            age_days = (time.time() - ep.created_at) / 86400
            lines.append(f"- [{age_days:.0f}d ago] {ep.summary}")
        return "\n".join(lines)

What to store: Not every message deserves to be an episode. Store turning points: decisions the user made, preferences stated, problems solved, and failures. A good heuristic: if you'd want to know this in the next conversation, store it. Use the importance field to weight retrieval — user-stated preferences score higher than routine exchanges.

Semantic Memory: Facts About Users and the World

Semantic memory stores structured facts independently of when they were learned. User preferences, attributes, and domain knowledge live here. Unlike episodic memory, semantic memory should be deduplicated and updated in place — you don't want five conflicting records about a user's timezone.

from dataclasses import dataclass
from enum import Enum
import time


class MemoryCategory(str, Enum):
    PREFERENCE = "preference"
    FACT = "fact"
    GOAL = "goal"
    CONSTRAINT = "constraint"


@dataclass
class SemanticFact:
    key: str                        # e.g., "user.timezone", "user.prefers_concise"
    value: Any
    category: MemoryCategory
    confidence: float = 1.0         # 0-1
    source: str = "explicit"        # "explicit" | "inferred"
    updated_at: float = field(default_factory=time.time)
    ttl_seconds: float | None = None  # None = permanent


class SemanticMemoryStore:
    def __init__(self):
        self._store: dict[str, dict[str, SemanticFact]] = {}  # user_id → key → fact

    def set(
        self,
        user_id: str,
        key: str,
        value: Any,
        category: MemoryCategory,
        confidence: float = 1.0,
        source: str = "explicit",
        ttl_seconds: float | None = None,
    ) -> SemanticFact:
        fact = SemanticFact(
            key=key, value=value, category=category,
            confidence=confidence, source=source, ttl_seconds=ttl_seconds,
        )
        if user_id not in self._store:
            self._store[user_id] = {}
        self._store[user_id][key] = fact
        return fact

    def get(self, user_id: str, key: str) -> SemanticFact | None:
        fact = self._store.get(user_id, {}).get(key)
        if fact is None:
            return None
        if fact.ttl_seconds is not None:
            if time.time() - fact.updated_at > fact.ttl_seconds:
                del self._store[user_id][key]
                return None
        return fact

    def get_all(self, user_id: str, category: MemoryCategory | None = None) -> list[SemanticFact]:
        facts = list(self._store.get(user_id, {}).values())
        now = time.time()
        valid = [f for f in facts if f.ttl_seconds is None or now - f.updated_at <= f.ttl_seconds]
        if category:
            valid = [f for f in valid if f.category == category]
        return valid

    def format_for_context(self, user_id: str) -> str:
        preferences = self.get_all(user_id, MemoryCategory.PREFERENCE)
        facts = self.get_all(user_id, MemoryCategory.FACT)
        constraints = self.get_all(user_id, MemoryCategory.CONSTRAINT)
        lines = []
        if preferences:
            lines.append("User preferences: " + "; ".join(f"{p.key}={p.value}" for p in preferences))
        if facts:
            lines.append("Known facts: " + "; ".join(f"{f.key}={f.value}" for f in facts))
        if constraints:
            lines.append("Constraints: " + "; ".join(f"{c.key}={c.value}" for c in constraints))
        return "\n".join(lines)

Extracting Semantic Facts from Conversations

You can run a lightweight extraction pass after each conversation turn to identify facts worth storing. This creates a feedback loop: conversations generate episodic memories, which are distilled into semantic facts over time.

EXTRACT_FACTS_PROMPT = """Analyze this conversation turn and extract any persistent facts about the user.

USER MESSAGE: {message}
ASSISTANT RESPONSE: {response}

Return a JSON array of facts to remember, or [] if none. Each fact:
{{
  "key": "user.",
  "value": ,
  "category": "preference|fact|goal|constraint",
  "confidence": <0.0 to 1.0>,
  "source": "explicit"
}}

Only extract facts the user explicitly stated or clearly implied. Do not infer preferences from a single message."""


def extract_semantic_facts(
    client: anthropic.Anthropic,
    user_message: str,
    assistant_response: str,
) -> list[dict]:
    result = client.messages.create(
        model="claude-haiku-4-5-20251001",
        max_tokens=512,
        messages=[{
            "role": "user",
            "content": EXTRACT_FACTS_PROMPT.format(
                message=user_message,
                response=assistant_response,
            )
        }],
    )
    import json
    try:
        return json.loads(result.content[0].text)
    except (json.JSONDecodeError, TypeError):
        return []

Procedural Memory: Learned Behaviors

Procedural memory stores how to do things — not facts, but patterns of successful action. For LLM agents, this maps to few-shot examples that encode what good responses look like for a given input type, and heuristics extracted from past successes and failures.

@dataclass
class ProceduralExample:
    task_type: str              # e.g., "summarize_meeting", "debug_python"
    input_pattern: str          # describes input characteristics
    input_example: str          # concrete input example
    output_example: str         # concrete good output
    embedding: list[float]
    outcome: str = "success"    # "success" | "failure"
    score: float = 1.0          # quality score from eval or user feedback
    created_at: float = field(default_factory=time.time)


class ProceduralMemoryStore:
    def __init__(self):
        self.examples: list[ProceduralExample] = []

    def store_example(
        self,
        task_type: str,
        input_example: str,
        output_example: str,
        embedding: list[float],
        score: float = 1.0,
        outcome: str = "success",
    ) -> ProceduralExample:
        ex = ProceduralExample(
            task_type=task_type,
            input_pattern=task_type,
            input_example=input_example,
            output_example=output_example,
            embedding=embedding,
            outcome=outcome,
            score=score,
        )
        self.examples.append(ex)
        return ex

    def retrieve_few_shot(
        self,
        query_embedding: list[float],
        task_type: str | None = None,
        k: int = 3,
        min_score: float = 0.7,
    ) -> list[ProceduralExample]:
        q = np.array(query_embedding)
        candidates = [
            e for e in self.examples
            if e.outcome == "success"
            and e.score >= min_score
            and (task_type is None or e.task_type == task_type)
        ]
        scored = []
        for ex in candidates:
            e = np.array(ex.embedding)
            sim = float(np.dot(q, e) / (np.linalg.norm(q) * np.linalg.norm(e) + 1e-8))
            scored.append((sim * ex.score, ex))
        scored.sort(reverse=True)
        return [ex for _, ex in scored[:k]]

    def format_few_shot(self, examples: list[ProceduralExample]) -> str:
        if not examples:
            return ""
        parts = ["Here are examples of good responses for similar requests:"]
        for i, ex in enumerate(examples, 1):
            parts.append(f"\nExample {i}:")
            parts.append(f"Input: {ex.input_example}")
            parts.append(f"Output: {ex.output_example}")
        return "\n".join(parts)

The MemoryManager: Composing All Three

In production you want a single interface that handles all memory types, assembles the right context for each turn, and enforces a token budget across memory sources.

@dataclass
class MemoryContext:
    episodic: str       # relevant past episodes
    semantic: str       # user facts and preferences
    procedural: str     # relevant few-shot examples
    total_tokens: int   # estimated token cost

    def as_system_addendum(self) -> str:
        parts = []
        if self.semantic:
            parts.append(self.semantic)
        if self.episodic:
            parts.append(self.episodic)
        if self.procedural:
            parts.append(self.procedural)
        return "\n\n".join(parts) if parts else ""


class MemoryManager:
    def __init__(
        self,
        client: anthropic.Anthropic,
        episodic: EpisodicMemoryStore,
        semantic: SemanticMemoryStore,
        procedural: ProceduralMemoryStore,
        token_budget: int = 2000,
    ):
        self.client = client
        self.episodic = episodic
        self.semantic = semantic
        self.procedural = procedural
        self.token_budget = token_budget

    def retrieve(
        self,
        user_id: str,
        session_id: str,
        current_message: str,
        task_type: str | None = None,
    ) -> MemoryContext:
        # Semantic: always include (small, structured)
        semantic_text = self.semantic.format_for_context(user_id)
        semantic_tokens = len(semantic_text.split()) * 1.3

        remaining = self.token_budget - semantic_tokens

        # Episodic: fill remaining budget (most recent wins)
        query_emb = self.episodic.embed(current_message)
        episodes = self.episodic.retrieve(current_message, user_id, k=8)
        episodic_text = self.episodic.format_for_context(episodes)
        episodic_tokens = len(episodic_text.split()) * 1.3
        remaining -= episodic_tokens

        # Procedural: only if budget remains
        procedural_text = ""
        if remaining > 200 and task_type:
            examples = self.procedural.retrieve_few_shot(query_emb, task_type, k=2)
            procedural_text = self.procedural.format_few_shot(examples)

        return MemoryContext(
            episodic=episodic_text,
            semantic=semantic_text,
            procedural=procedural_text,
            total_tokens=int(self.token_budget - remaining),
        )

    def after_turn(
        self,
        user_id: str,
        session_id: str,
        user_message: str,
        assistant_response: str,
    ) -> None:
        """Run after each turn to update memory stores."""
        # Store episode
        self.episodic.store(
            user_id=user_id,
            session_id=session_id,
            content=f"User: {user_message}\nAssistant: {assistant_response}",
        )
        # Extract semantic facts
        facts = extract_semantic_facts(self.client, user_message, assistant_response)
        for fact in facts:
            self.semantic.set(
                user_id=user_id,
                key=fact["key"],
                value=fact["value"],
                category=MemoryCategory(fact["category"]),
                confidence=fact.get("confidence", 0.8),
                source=fact.get("source", "explicit"),
            )

Memory Decay and Eviction

Memory that never decays grows without bound and eventually degrades retrieval quality — old, irrelevant episodes drown out recent relevant ones. Three patterns for managing this:

Strategy Mechanism Use When
TTL expiry Set ttl_seconds on semantic facts; evict on read Preferences that change (e.g., "working on project X") — days to weeks
Importance decay Multiply importance by 0.99 per day; cull below threshold Episodic memories — recency matters, old episodes fade
Consolidation Nightly job: cluster similar episodes, replace cluster with summary High-volume users with thousands of episodes — compress to themes 
def apply_time_decay(episodes: list[Episode], half_life_days: float = 30.0) -> None:
    """Decay importance of episodes based on age. Modifies in place."""
    now = time.time()
    half_life_s = half_life_days * 86400
    for ep in episodes:
        age_s = now - ep.created_at
        ep.importance *= (0.5 ** (age_s / half_life_s))


def consolidate_episodes(
    client: anthropic.Anthropic,
    episodes: list[Episode],
    min_cluster_size: int = 5,
) -> list[Episode]:
    """Replace dense clusters of similar episodes with summaries."""
    # Group by rough similarity (simplified — use proper clustering in production)
    if len(episodes) < min_cluster_size:
        return episodes

    combined = "\n".join(ep.summary for ep in episodes)
    result = client.messages.create(
        model="claude-haiku-4-5-20251001",
        max_tokens=200,
        messages=[{
            "role": "user",
            "content": f"Synthesize these {len(episodes)} memories into one summary:\n{combined}"
        }],
    )
    consolidated = Episode(
        id=episode_id(episodes[0].user_id, "consolidated"),
        session_id="consolidated",
        user_id=episodes[0].user_id,
        content=combined,
        summary=result.content[0].text,
        embedding=episodes[0].embedding,  # Use centroid in production
        metadata={"consolidated_from": len(episodes)},
        importance=max(ep.importance for ep in episodes),
    )
    return [consolidated]

Memory vs. state: Don't conflate memory with session state. Session state (current task, tool call results, in-progress work) belongs in the context window and should be discarded at session end. Memory is what persists across sessions — things worth knowing next time. Mixing them leads to bloated memory stores full of transient data.

Storage Backend Selection

The choice of storage backend matters at scale:

  • Episodic memory: Pinecone, Weaviate, Qdrant, or pgvector (PostgreSQL). For most teams, pgvector is the right starting point — no additional infrastructure, SQL for metadata filtering, vector search for retrieval.
  • Semantic memory: PostgreSQL or Redis with JSON support. Structured data with known keys; vector search is rarely needed for well-keyed facts.
  • Procedural memory: Same vector DB as episodic, different collection/namespace. Few-shot examples benefit from embedding-based retrieval over task type.

The right architecture for most teams starting out: pgvector for episodic and procedural, PostgreSQL row store for semantic. Migrate to a dedicated vector DB when you hit millions of episodes or need sub-10ms retrieval at scale.
