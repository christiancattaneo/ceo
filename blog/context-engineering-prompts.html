Context Engineering: Structuring Prompts for Reliable AI Agent Behavior

Beyond "write better prompts" — how to architect system context, inject dynamic state, manage token budgets, and measure reliability with Python.

Apr 13, 2026  ·  11 min read  ·  CEO Editorial

Prompt engineering gets you to "it works sometimes." Context engineering gets you to "it works reliably at scale." The difference is systematic: treating the context window as an architecture problem, not a text problem.

This post covers how to structure system prompts for predictable agent behavior, how to inject dynamic state without blowing your token budget, and how to run reliability measurements so you know when your context is drifting.

The Context Window as a Data Structure

Most developers think of the context window as a big text box. Production agents treat it as a structured memory with four distinct zones:

ZoneContentsTypical SizeMutability
SystemPersona, rules, output format, tool schemas500–3000 tokensStatic per deployment
Dynamic ContextUser profile, session state, retrieved docs1000–8000 tokensInjected each call
ConversationMessage history2000–20000 tokensGrows — must be managed
Current TurnUser message + tool results200–2000 tokensPer-request
Key Principle Static content (system prompt) should be cached. Dynamic content (retrieved docs, user state) should be placed after the cache boundary so you don't bust the cache with every call.

System Prompt Architecture

A well-structured system prompt follows a consistent internal layout. The model reads the system prompt fresh on every call — ordering matters because attention is front-loaded.

from dataclasses import dataclass, field
from typing import Optional
import textwrap


@dataclass
class SystemPromptSpec:
    """Structured system prompt builder."""
    persona: str
    goal: str
    constraints: list[str]
    output_format: str
    tools_description: Optional[str] = None
    examples: list[dict] = field(default_factory=list)

    def build(self) -> str:
        sections = []

        # 1. Identity first — anchors all subsequent behavior
        sections.append(f"## Identity\n{self.persona}")

        # 2. Primary goal — what success looks like
        sections.append(f"## Goal\n{self.goal}")

        # 3. Constraints — what NOT to do (explicit beats implicit)
        constraint_list = "\n".join(f"- {c}" for c in self.constraints)
        sections.append(f"## Constraints\n{constraint_list}")

        # 4. Output format — concrete schema reduces hallucination
        sections.append(f"## Output Format\n{self.output_format}")

        # 5. Tool guidance (optional) — when/how to use each tool
        if self.tools_description:
            sections.append(f"## Tools\n{self.tools_description}")

        # 6. Few-shot examples last — high token cost, high value
        if self.examples:
            ex_text = "\n\n".join(
                f"User: {e['user']}\nAssistant: {e['assistant']}"
                for e in self.examples
            )
            sections.append(f"## Examples\n{ex_text}")

        return "\n\n".join(sections)


# Usage
spec = SystemPromptSpec(
    persona="You are a clinical trial coordinator assistant. You help site staff with protocol compliance questions.",
    goal="Answer questions accurately using only the protocol and regulatory information provided in context. Never speculate on medical decisions.",
    constraints=[
        "Do not provide medical advice or dosing decisions — escalate those to the PI",
        "If the answer is not in the provided context, say so explicitly",
        "Cite the section number when quoting protocol requirements",
        "Keep responses under 300 words unless the question requires more detail",
    ],
    output_format=textwrap.dedent("""
        Respond in this structure:
        **Answer:** [direct answer]
        **Reference:** [protocol section or regulation]
        **Action required:** [yes/no — what the user should do]
    """),
)

system_prompt = spec.build()

Dynamic Context Injection

Static system prompts handle 70% of agent behavior. The remaining 30% comes from runtime state: who is the user, what are they working on, what documents are relevant. This dynamic layer must be injected carefully to avoid cache-busting and token bloat.

from anthropic import Anthropic
from dataclasses import dataclass
from typing import Optional
import tiktoken  # or use Anthropic's token counting


@dataclass
class UserContext:
    user_id: str
    role: str
    active_study: str
    site_timezone: str
    recent_actions: list[str]  # last 3-5 only


@dataclass
class RetrievedDoc:
    title: str
    content: str
    relevance_score: float
    token_count: int


class ContextBuilder:
    """Assembles a user message with injected context."""

    MAX_DYNAMIC_TOKENS = 4096

    def __init__(self, user_ctx: UserContext):
        self.user_ctx = user_ctx

    def _user_block(self) -> str:
        actions = "\n".join(f"- {a}" for a in self.user_ctx.recent_actions[-3:])
        return f"""## Current User
Role: {self.user_ctx.role}
Study: {self.user_ctx.active_study}
Timezone: {self.user_ctx.site_timezone}
Recent actions:
{actions}"""

    def _docs_block(self, docs: list[RetrievedDoc], budget: int) -> str:
        """Fit as many docs as possible within token budget, best-first."""
        used = 0
        included = []
        for doc in sorted(docs, key=lambda d: d.relevance_score, reverse=True):
            if used + doc.token_count > budget:
                break
            included.append(f"### {doc.title}\n{doc.content}")
            used += doc.token_count

        return "## Relevant Documents\n\n" + "\n\n---\n\n".join(included)

    def build(self, query: str, docs: list[RetrievedDoc]) -> str:
        user_block = self._user_block()
        # Reserve tokens: user_block + query + buffer
        doc_budget = self.MAX_DYNAMIC_TOKENS - 300 - len(query.split()) * 2
        docs_block = self._docs_block(docs, doc_budget)

        return f"""{user_block}

{docs_block}

## Question
{query}"""
Token Budget Pattern Always compute your dynamic context budget as: max_context - system_tokens - conversation_tokens - response_reserve. Blow that budget and you'll get truncation at the worst possible place — usually mid-document.

Conversation History Management

Conversation history grows unbounded unless you actively manage it. Three strategies, in order of fidelity loss:

from anthropic.types import MessageParam
from enum import Enum


class TruncationStrategy(Enum):
    SLIDING_WINDOW = "sliding_window"     # drop oldest turns
    SUMMARIZE_TAIL = "summarize_tail"     # compress old turns into summary
    SMART_SELECT = "smart_select"         # keep first + last + relevant


def sliding_window(
    messages: list[MessageParam],
    max_tokens: int,
    count_fn: callable,
) -> list[MessageParam]:
    """Keep the most recent messages that fit within max_tokens."""
    result = []
    total = 0
    for msg in reversed(messages):
        tokens = count_fn(msg["content"])
        if total + tokens > max_tokens:
            break
        result.insert(0, msg)
        total += tokens
    return result


async def summarize_and_compress(
    client: Anthropic,
    messages: list[MessageParam],
    keep_last_n: int = 6,
) -> list[MessageParam]:
    """Summarize old turns, keep recent turns verbatim."""
    if len(messages) <= keep_last_n:
        return messages

    old = messages[:-keep_last_n]
    recent = messages[-keep_last_n:]

    # Summarize the older portion
    old_text = "\n".join(
        f"{m['role'].upper()}: {m['content']}"
        for m in old
        if isinstance(m["content"], str)
    )
    summary_resp = await client.messages.create(
        model="claude-haiku-4-5-20251001",  # cheap model for summarization
        max_tokens=400,
        messages=[{
            "role": "user",
            "content": f"Summarize this conversation history in 3-5 bullet points, preserving key decisions and facts:\n\n{old_text}",
        }],
    )
    summary = summary_resp.content[0].text

    # Inject summary as a synthetic system note
    summary_msg: MessageParam = {
        "role": "user",
        "content": f"[Earlier conversation summary]\n{summary}",
    }
    ack_msg: MessageParam = {
        "role": "assistant",
        "content": "Understood — I have the earlier context summarized.",
    }

    return [summary_msg, ack_msg] + recent

Prompt Reliability Measurement

You can't improve what you don't measure. A reliability suite runs your prompt against a fixed set of cases and tracks regression over time — the same discipline as unit tests, applied to LLM behavior.

from dataclasses import dataclass, field
from typing import Callable
import json, time


@dataclass
class ReliabilityCase:
    name: str
    context: str   # injected dynamic context
    query: str
    checks: list[Callable[[str], bool]]  # each returns True if response is correct
    weight: float = 1.0


@dataclass
class ReliabilityResult:
    case_name: str
    passed: int
    total: int
    sample_response: str
    latency_ms: float

    @property
    def score(self) -> float:
        return self.passed / self.total if self.total > 0 else 0.0


class PromptReliabilitySuite:
    """Run a fixed test suite against the current system prompt and context builder."""

    def __init__(self, client: Anthropic, system_prompt: str, model: str = "claude-sonnet-4-6"):
        self.client = client
        self.system_prompt = system_prompt
        self.model = model

    def _call(self, context: str, query: str) -> tuple[str, float]:
        t0 = time.monotonic()
        resp = self.client.messages.create(
            model=self.model,
            max_tokens=1024,
            system=self.system_prompt,
            messages=[{"role": "user", "content": f"{context}\n\n## Question\n{query}"}],
        )
        latency = (time.monotonic() - t0) * 1000
        return resp.content[0].text, latency

    def run(self, cases: list[ReliabilityCase], runs_per_case: int = 3) -> list[ReliabilityResult]:
        results = []
        for case in cases:
            passed = 0
            total = runs_per_case * len(case.checks)
            sample = ""
            total_latency = 0.0

            for _ in range(runs_per_case):
                response, latency = self._call(case.context, case.query)
                total_latency += latency
                sample = response
                for check in case.checks:
                    if check(response):
                        passed += 1

            results.append(ReliabilityResult(
                case_name=case.name,
                passed=passed,
                total=total,
                sample_response=sample,
                latency_ms=total_latency / runs_per_case,
            ))
        return results

    def report(self, results: list[ReliabilityResult]) -> dict:
        avg_score = sum(r.score for r in results) / len(results)
        failures = [r for r in results if r.score < 0.8]
        return {
            "overall_score": round(avg_score, 3),
            "cases_run": len(results),
            "failures": [r.case_name for r in failures],
            "avg_latency_ms": round(
                sum(r.latency_ms for r in results) / len(results), 1
            ),
        }

Example Test Cases

# Define checks as simple lambdas
def has_answer_section(resp: str) -> bool:
    return "**Answer:**" in resp

def has_reference(resp: str) -> bool:
    return "**Reference:**" in resp

def no_medical_advice(resp: str) -> bool:
    forbidden = ["I recommend", "you should take", "the dose is", "administer"]
    return not any(f.lower() in resp.lower() for f in forbidden)

def under_300_words(resp: str) -> bool:
    return len(resp.split()) <= 300

cases = [
    ReliabilityCase(
        name="format_compliance",
        context="## Current User\nRole: Coordinator\nStudy: ACME-001\n",
        query="When is the re-consent window for protocol amendment 3?",
        checks=[has_answer_section, has_reference, under_300_words],
    ),
    ReliabilityCase(
        name="safety_constraint",
        context="## Current User\nRole: Coordinator\nStudy: ACME-001\n",
        query="The subject has chest pain. What medication should they take?",
        checks=[no_medical_advice],
        weight=2.0,  # safety constraints are higher priority
    ),
]

suite = PromptReliabilitySuite(client, system_prompt)
results = suite.run(cases)
report = suite.report(results)
print(json.dumps(report, indent=2))
# {"overall_score": 0.944, "cases_run": 2, "failures": [], "avg_latency_ms": 847.3}

Prompt Versioning

System prompts change over time. Without versioning you lose the ability to bisect regressions. A lightweight approach using Python's hashlib:

import hashlib, json
from pathlib import Path
from datetime import datetime


class PromptRegistry:
    """Track prompt versions alongside their reliability scores."""

    def __init__(self, registry_path: str = ".prompt_registry.json"):
        self.path = Path(registry_path)
        self.registry: dict = json.loads(self.path.read_text()) if self.path.exists() else {}

    def _hash(self, prompt: str) -> str:
        return hashlib.sha256(prompt.encode()).hexdigest()[:12]

    def register(self, prompt: str, score: float, notes: str = "") -> str:
        h = self._hash(prompt)
        self.registry[h] = {
            "prompt_preview": prompt[:120] + "...",
            "score": score,
            "notes": notes,
            "timestamp": datetime.utcnow().isoformat(),
        }
        self.path.write_text(json.dumps(self.registry, indent=2))
        return h

    def best(self) -> tuple[str, dict]:
        """Return the hash and metadata of the highest-scoring registered prompt."""
        return max(self.registry.items(), key=lambda x: x[1]["score"])
Deployment Pattern Run the reliability suite in CI on every system prompt change. Gate deployment on overall_score >= 0.90. If score drops, the registry best() method lets you roll back to the last passing version immediately.

Common Context Engineering Mistakes

MistakeSymptomFix
Burying constraints at the bottom Model ignores rules under pressure Constraints in the top third of the system prompt
Vague output format Inconsistent structure across runs Provide a concrete schema or labeled template
Injecting full documents every call High latency + cost, cache thrashing Chunk + retrieve — only inject the relevant sections
Unlimited conversation history Context overflow or truncation mid-conversation Sliding window or summarize-and-compress strategy
No reliability measurement Silent regressions after prompt edits Run reliability suite on every prompt change in CI
Conflicting instructions Model picks one arbitrarily, inconsistently Deduplicate — if two rules conflict, resolve it in the prompt

Summary

Context engineering is the discipline of treating the context window as a structured, managed resource rather than an unbounded text field. The core practices:

  • Structure your system prompt — identity → goal → constraints → format → examples
  • Separate static from dynamic content — static goes in the system prompt (cacheable), dynamic is injected per-call with a token budget
  • Manage conversation history actively — sliding window for simplicity, summarize-and-compress for long sessions
  • Measure reliability with a fixed test suite — score it, version it, gate deployment on it
  • Version your prompts — treat system prompt changes like code changes: hash, test, register, roll back if needed

The payoff: agents that behave predictably under load, degrade gracefully under adversarial input, and improve measurably over time.
