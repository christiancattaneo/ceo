Async Python Patterns for LLM Applications

Concurrent API calls, streaming responses, async agent pipelines, semaphore-based rate limiting, and timeout handling — with full working code.

Apr 13, 2026  ·  12 min read  ·  CEO Editorial

LLM API calls are slow by the standards of most I/O — a typical Claude response takes 1–10 seconds. Run them synchronously and you serialize all that latency. Run them with asyncio and you can fire dozens concurrently, stream tokens to the user as they arrive, and build agent pipelines that process work in parallel.

This post covers the async patterns that actually matter for production LLM apps: concurrent calls with proper rate-limit backpressure, streaming with async for, async queues for agent task distribution, and cancellation/timeout handling that won't leak resources.

asyncio Fundamentals for LLM Work

Before the patterns: the three primitives you'll use constantly.

import asyncio
from anthropic import AsyncAnthropic

client = AsyncAnthropic()  # async client — not the sync Anthropic()


# 1. Basic async call
async def single_call(prompt: str) -> str:
    msg = await client.messages.create(
        model="claude-sonnet-4-6",
        max_tokens=1024,
        messages=[{"role": "user", "content": prompt}],
    )
    return msg.content[0].text


# 2. gather — run N coroutines concurrently, collect all results
async def parallel_calls(prompts: list[str]) -> list[str]:
    tasks = [single_call(p) for p in prompts]
    return await asyncio.gather(*tasks)


# 3. TaskGroup (Python 3.11+) — structured concurrency with automatic cleanup
async def parallel_with_taskgroup(prompts: list[str]) -> list[str]:
    results = [None] * len(prompts)
    async with asyncio.TaskGroup() as tg:
        for i, prompt in enumerate(prompts):
            async def _run(idx=i, p=prompt):
                results[idx] = await single_call(p)
            tg.create_task(_run())
    return results
gather vs TaskGroup gather is simpler but if one task raises, others keep running (unless return_exceptions=False). TaskGroup cancels all sibling tasks on the first exception — safer for production pipelines where a partial result is useless.

Semaphore-Based Rate Limiting

Firing 50 concurrent Claude calls will hit rate limits. A semaphore bounds concurrency to a safe number while keeping everything async:

import asyncio, time
from anthropic import AsyncAnthropic
from dataclasses import dataclass


@dataclass
class RateLimitedClient:
    """Async Claude client with concurrency + token-rate limits."""
    max_concurrent: int = 5
    requests_per_minute: int = 50

    def __post_init__(self):
        self._client = AsyncAnthropic()
        self._sem = asyncio.Semaphore(self.max_concurrent)
        self._request_times: list[float] = []

    async def _wait_for_rate_limit(self):
        """Sliding window rate limiter."""
        now = time.monotonic()
        window = 60.0
        # Drop timestamps outside the window
        self._request_times = [t for t in self._request_times if now - t < window]

        if len(self._request_times) >= self.requests_per_minute:
            oldest = self._request_times[0]
            sleep_for = window - (now - oldest) + 0.1
            if sleep_for > 0:
                await asyncio.sleep(sleep_for)

        self._request_times.append(time.monotonic())

    async def call(self, prompt: str, model: str = "claude-sonnet-4-6", max_tokens: int = 1024) -> str:
        async with self._sem:  # blocks when max_concurrent is reached
            await self._wait_for_rate_limit()
            msg = await self._client.messages.create(
                model=model,
                max_tokens=max_tokens,
                messages=[{"role": "user", "content": prompt}],
            )
            return msg.content[0].text

    async def batch(self, prompts: list[str], **kwargs) -> list[str]:
        """Process all prompts concurrently, respecting rate limits."""
        tasks = [self.call(p, **kwargs) for p in prompts]
        return await asyncio.gather(*tasks, return_exceptions=True)


# Process 20 prompts concurrently, max 5 at a time, max 50/min
async def main():
    client = RateLimitedClient(max_concurrent=5, requests_per_minute=50)
    prompts = [f"Summarize topic {i} in one sentence." for i in range(20)]
    results = await client.batch(prompts)
    for r in results:
        if isinstance(r, Exception):
            print(f"Error: {r}")
        else:
            print(r)

Streaming Responses

Streaming lets you display tokens to users as they arrive instead of waiting for the full response. The Anthropic async client supports streaming via an async context manager:

from anthropic import AsyncAnthropic

client = AsyncAnthropic()


async def stream_to_stdout(prompt: str) -> str:
    """Stream tokens to stdout and return the full response."""
    full_text = ""
    async with client.messages.stream(
        model="claude-sonnet-4-6",
        max_tokens=2048,
        messages=[{"role": "user", "content": prompt}],
    ) as stream:
        async for text in stream.text_stream:
            print(text, end="", flush=True)
            full_text += text
    print()  # newline after stream
    return full_text


# FastAPI streaming endpoint
from fastapi import FastAPI
from fastapi.responses import StreamingResponse

app = FastAPI()

@app.post("/chat")
async def chat(body: dict):
    prompt = body.get("prompt", "")

    async def token_generator():
        async with client.messages.stream(
            model="claude-sonnet-4-6",
            max_tokens=2048,
            messages=[{"role": "user", "content": prompt}],
        ) as stream:
            async for text in stream.text_stream:
                # Server-Sent Events format
                yield f"data: {text}\n\n"
        yield "data: [DONE]\n\n"

    return StreamingResponse(token_generator(), media_type="text/event-stream")
Streaming + Tool Use Streaming works with tool-calling models. Listen for stream.current_message_snapshot events to detect when the model starts a tool call mid-stream. The Anthropic SDK exposes these as typed event objects — check for content_block_start events with type == "tool_use".

Async Queue-Based Agent Pipelines

When you have a pipeline where one LLM call feeds the next, or where multiple worker agents pull from a shared task queue, asyncio.Queue gives you a clean coordination primitive:

import asyncio
from dataclasses import dataclass
from typing import Any
from anthropic import AsyncAnthropic


@dataclass
class Task:
    task_id: str
    prompt: str
    metadata: dict = None


@dataclass
class Result:
    task_id: str
    output: str
    error: Exception | None = None


async def worker(
    worker_id: int,
    task_queue: asyncio.Queue[Task | None],
    result_queue: asyncio.Queue[Result],
    client: AsyncAnthropic,
):
    """Pull tasks from queue, call Claude, push results."""
    while True:
        task = await task_queue.get()
        if task is None:  # sentinel — this worker should stop
            task_queue.task_done()
            break

        try:
            msg = await client.messages.create(
                model="claude-haiku-4-5-20251001",  # fast/cheap for workers
                max_tokens=512,
                messages=[{"role": "user", "content": task.prompt}],
            )
            result = Result(task_id=task.task_id, output=msg.content[0].text)
        except Exception as e:
            result = Result(task_id=task.task_id, output="", error=e)
        finally:
            task_queue.task_done()

        await result_queue.put(result)


async def run_pipeline(tasks: list[Task], num_workers: int = 4) -> list[Result]:
    client = AsyncAnthropic()
    task_q: asyncio.Queue = asyncio.Queue()
    result_q: asyncio.Queue = asyncio.Queue()

    # Enqueue all tasks
    for task in tasks:
        await task_q.put(task)

    # Send one sentinel per worker to signal shutdown
    for _ in range(num_workers):
        await task_q.put(None)

    # Spawn workers
    worker_tasks = [
        asyncio.create_task(worker(i, task_q, result_q, client))
        for i in range(num_workers)
    ]

    # Wait for all workers to finish
    await asyncio.gather(*worker_tasks)

    # Collect results
    results = []
    while not result_q.empty():
        results.append(await result_q.get())

    return results

Timeout and Cancellation

LLM calls can hang. Always wrap long-running calls in a timeout, and understand how cancellation propagates through your async call stack:

import asyncio
from anthropic import AsyncAnthropic

client = AsyncAnthropic()


async def call_with_timeout(
    prompt: str,
    timeout_seconds: float = 30.0,
    fallback: str = "[Response timed out]",
) -> str:
    try:
        return await asyncio.wait_for(
            client.messages.create(
                model="claude-sonnet-4-6",
                max_tokens=1024,
                messages=[{"role": "user", "content": prompt}],
            ),
            timeout=timeout_seconds,
        )
    except asyncio.TimeoutError:
        return fallback
    except asyncio.CancelledError:
        # Task was cancelled by a parent — re-raise to propagate correctly
        raise


# Python 3.11+ — asyncio.timeout() context manager (preferred)
async def call_with_deadline(prompt: str, deadline_seconds: float = 30.0) -> str:
    try:
        async with asyncio.timeout(deadline_seconds):
            msg = await client.messages.create(
                model="claude-sonnet-4-6",
                max_tokens=1024,
                messages=[{"role": "user", "content": prompt}],
            )
            return msg.content[0].text
    except TimeoutError:
        return "[Deadline exceeded]"


# Cancellable long-running agent — can be cancelled from outside
class CancellableAgent:
    def __init__(self):
        self._task: asyncio.Task | None = None

    async def run(self, steps: list[str]) -> list[str]:
        """Run a multi-step pipeline. Can be cancelled mid-flight."""
        results = []
        for step in steps:
            try:
                result = await call_with_deadline(step)
                results.append(result)
            except asyncio.CancelledError:
                # Clean up any open connections here if needed
                raise  # must re-raise CancelledError — never swallow it
        return results

    def start(self, steps: list[str]) -> asyncio.Task:
        self._task = asyncio.create_task(self.run(steps))
        return self._task

    def cancel(self):
        if self._task and not self._task.done():
            self._task.cancel()
Never Swallow CancelledError asyncio.CancelledError is how Python's event loop signals a task to stop. If you catch it and don't re-raise, the task appears stuck — it won't cancel and its parent awaiter will hang. Always re-raise after cleanup.

Async-Safe State Management

Multiple concurrent coroutines mutating shared state is a race condition waiting to happen. Use asyncio.Lock to protect shared resources:

import asyncio
from collections import defaultdict


class ConversationStore:
    """Thread-safe (async-safe) in-memory conversation history."""

    def __init__(self):
        self._histories: dict[str, list] = defaultdict(list)
        self._locks: dict[str, asyncio.Lock] = defaultdict(asyncio.Lock)

    async def append(self, session_id: str, role: str, content: str):
        async with self._locks[session_id]:
            self._histories[session_id].append({"role": role, "content": content})

    async def get(self, session_id: str) -> list:
        async with self._locks[session_id]:
            return list(self._histories[session_id])  # return copy

    async def chat(self, session_id: str, user_message: str) -> str:
        await self.append(session_id, "user", user_message)
        history = await self.get(session_id)

        msg = await client.messages.create(
            model="claude-sonnet-4-6",
            max_tokens=1024,
            messages=history,
        )
        response_text = msg.content[0].text
        await self.append(session_id, "assistant", response_text)
        return response_text

When to Use sync vs. async

ScenarioUseReason
Single call in a script Sync Anthropic() asyncio overhead not worth it for one call
Batch of 5+ calls Async AsyncAnthropic() + gather Concurrent execution cuts wall time dramatically
FastAPI / Starlette endpoint Async — always Sync calls block the event loop and starve other requests
Streaming to browser Async + StreamingResponse Sync streaming blocks server; async streams tokens per chunk
Celery / background workers Sync in worker + asyncio.run() if needed Celery workers are synchronous by default; wrap async in run()
Multi-agent pipelines Async queues + TaskGroup Queue decouples producers from consumers; TaskGroup cleans up on failure

Summary

The async patterns for LLM apps reduce to a short list:

  • AsyncAnthropic() + gather for parallel batches
  • Semaphore + sliding window for rate-limit-safe concurrency
  • messages.stream() + async for for token-by-token streaming
  • asyncio.Queue + worker pool for agent task distribution
  • asyncio.timeout() for deadline enforcement — always re-raise CancelledError
  • asyncio.Lock per session for safe concurrent state mutation

Apply these consistently and LLM latency stops being a bottleneck — it becomes a scheduling problem you've already solved.
