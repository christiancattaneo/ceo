# X and ClawHub distribution for OpenClaw agents

This guide turns positioning into repeatable distribution.

## Core message architecture

Use one argument with two contrasts.

- **contrast 1**: citation visibility vs agent actionability
- **contrast 2**: content layer vs protocol layer

Anchor sentence:

`GEO gets you mentioned. CEO gets you called.`

## X profile and card setup

- use one canonical site URL in profile bio
- keep `twitter:card=summary_large_image`
- use one branded social image with readable headline text
- keep description under 200 chars

## Thread template

Structure each thread like this.

1. claim in one line
2. evidence from one benchmark or teardown
3. mechanism explanation in plain language
4. implementation checklist
5. single CTA to waitlist or demo URL

## Post ideas that attract operators

- teardown: "why this agent cannot call your API yet"
- benchmark: "scan 50 products for llms and endpoint readiness"
- migration: "from citation-only to call-ready in 48 hours"
- security: "how to expose agent actions without opening abuse vectors"

## ClawHub listing hygiene

- stable capability naming across all docs
- clear version and compatibility notes
- explicit auth requirements and safe scopes
- explicit failure states and retry semantics
- changelog entry for every capability behavior change

## Weekly operating loop

- Monday: publish benchmark or teardown
- Wednesday: post implementation excerpt and code proof
- Friday: publish short scorecard with wins and misses

Keep one source-of-truth page updated every week so posts always point to fresh state.
